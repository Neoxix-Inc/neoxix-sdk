# Security Policy

## Supported versions

The project is in early development. Security fixes are provided only for the latest code on `main` and the most recent published version, if one exists.

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in public issues, discussions, pull requests, or logs.

Use GitHub private vulnerability reporting from the repository **Security** tab when it is enabled. If that option is unavailable, contact a verified repository or organization maintainer through a private GitHub channel and ask for a secure reporting path. This project does not publish an unverified security email address.

Include a concise impact description, affected versions, reproduction steps, and a sanitized proof of concept. Do not include real credentials or production data. Maintainers will coordinate validation and disclosure through the private report.
